36 /// Information about stack frame layout on the target. It holds the direction
37 /// of stack growth, the known stack alignment on entry to each function, and
38 /// the offset to the locals area.
39 ///
40 /// The offset to the local area is the offset from the stack pointer on
41 /// function entry to the first location where function data (local variables,
42 /// spill locations) can be stored.

46  StackGrowsUp, // Adding to the stack increases the stack address
47  StackGrowsDown // Adding to the stack decreases the stack address
48  };

282  /// hasReservedCallFrame - Under normal circumstances, when a frame pointer is
283  /// not required, we reserve argument space for call sites in the function
284  /// immediately on entry to the current function. This eliminates the need for
285  /// add/sub sp brackets around call sites. Returns true if the call frame is
286  /// included as part of the stack frame.
287  virtual bool hasReservedCallFrame(const MachineFunction &MF) const {
288  return !hasFP(MF);

291  /// canSimplifyCallFramePseudos - When possible, it's best to simplify the
292  /// call frame pseudo ops before doing frame index elimination. This is
293  /// possible only when frame index references between the pseudos won't
294  /// need adjusting for the call frame adjustments. Normally, that's true
295  /// if the function has a reserved call frame or a frame pointer. Some
296  /// targets (Thumb2, for example) may have more complicated criteria,
297  /// however, and can override this behavior.
298  virtual bool canSimplifyCallFramePseudos(const MachineFunction &MF) const {
299  return hasReservedCallFrame(MF) || hasFP(MF);
